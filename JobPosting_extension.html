Extension for modelling job markets

Collection of vocabulary terms extending Schema.org's JobPosting to cover the domain of job market.

See wiki for details.

Compensation

Compensation describes how the worker is paid for the work done.

Subclass of: QuantitativeValue

Example (RDFa):

<p rel="compensation">
  Wage:
  <span property="currency" content="EUR">&euro;</span><span property="value">4.50</span> 
  (<span property="type">hourly</span> rate)
</p>

currency

Currency (in 3-letter ISO 4217 format), in which compensation is paid.

Domain: Compensation

Range: Text

Example (RDFa):

<span property="currency" content="EUR">&euro;</span>

type

Type of compensation. Recommended values are: annual, monthly, hourly, fixed.

Domain: Compensation

Range: Text

Example (RDFa):

<span property="type">hourly</span> rate

available vacancies

Number of persons sought for a job. Default value is 1.

Domain: JobPosting

Range: Integer

Example (RDFa):

<tr>
  <th>Number of open vacancies:</th>
  <td property="availableVacancies">5</td>
</tr>

compensation

Compensation paid by employer to employee as stated in job posting

Domain: JobPosting

Range: Compensation

Example (RDFa):

<p rel="compensation">
  Wage:
  <span property="value">4.50</span> <span property="currency" content="EUR">&euro;</span>
  (<span property="type">hourly</span> rate)
</p>

contact

Contact information job seeker needs to respond to job posting

Domain: JobPosting

Range: ContactPoint

Example (RDFa):

<div rel="contact">
  <p property="name">John Doe</p>
  <p>Phone: <span property="telephone" content="+35312345678">12 345678</span></p>
</div>

employer

Property relating job postings to persons or organizations offering a job

Domain: JobPosting

Range: Organization, Person

Example (RDFa):

<p>Employer: <a rel="employer" href="http://acme.com/">ACME Inc.</a></p>

experience requirements

Description of skills and experience needed for the position.

Domain: JobPosting

Range: Text, WorkExperience

Example (RDFa):

<div rel="experienceRequirements">
  <p>Required experience:</p>
  <p>
    At least a <span property="duration" content="P1Y">year</span> of
    <span property="description">work with forklift</span>
    (<span property="references" content="true">job applicant has to provide references</span>)
  </p>
</div>

is remote work

Indicates job postings for which location of execution does not matter (i.e. remote work)

Domain: JobPosting

Range: Boolean

Example (RDFa):

<p property="isRemoteWork" content="true">Work from home.</p>

start date

Date when employment starts

Domain: JobPosting

Range: Date

Example (RDFa):

<p>Start date: <span property="startDate" content="2013-04-01">April 1, 2013</span></p>

Work experience

Job applicant's work history that demonstrates skills or competences

Subclass of: StructuredValue

Example (RDFa):

<h2>Required experience:</h2>
<ul>
  <li rel="experienceRequirements"><span property="description">Forklift operation</span>, <span property="duration" content="P1Y">1 year</span></li>
  <li rel="experienceRequirements">
    <span property="description">Industry experience</span>,
    <span property="duration" content="P2Y">2 years</span>
    (<span property="references" content="true">job applicant has to provide references</span>)
  </li>
</ul>

duration

Length of period during which job applicant acquired experience

Domain: WorkExperience

Range: Duration

Example (RDFa):

<p>
  Job applicant must have at least a <span property="duration" content="P1Y">year</span> of experience in the field.
</p>

references

Indicates whether applicant is able to provide contact for references

Domain: WorkExperience

Range: Boolean

Example (RDFa):

<p>
  Previous work history need to be supported <span property="references" content="true">with references</span>.
</p>
